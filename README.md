# Система бронирования столиков в корпоративной столовой

Django веб-приложение для бронирования столиков и оформления предзаказов блюд.

## Технологии

- Django 3.2+
- PostgreSQL 12+
- Django Admin

## Установка и настройка

### 1. Установка зависимостей

```bash
# Активация виртуального окружения
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows

# Установка зависимостей
pip install -r requirements.txt
```

### 2. Настройка базы данных

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE restaurant_booking;
```

Настройте подключение через переменные окружения или отредактируйте `restaurant_booking/settings.py`:

```bash
export DB_NAME=restaurant_booking
export DB_USER=postgres
export DB_PASSWORD=postgres
export DB_HOST=localhost
export DB_PORT=5432
```

### 3. Применение миграций

```bash
python manage.py migrate
```

### 4. Создание суперпользователя

```bash
python manage.py createsuperuser
```

### 5. Запуск сервера разработки

```bash
python manage.py runserver
```

Административная панель будет доступна по адресу: http://127.0.0.1:8000/admin/

## Структура базы данных

### Модели

1. **Client (Клиенты)**
   - ФИО, email (уникальный), дата создания
   - Нельзя удалить при наличии активных бронирований

2. **Table (Столики)**
   - Номер столика (уникальный), количество мест
   - Нельзя удалить при наличии активных бронирований

3. **Dish (Блюда)**
   - Название, изображение, описание, доступное количество
   - Нельзя удалить при использовании в бронированиях

4. **Reservation (Бронирования)**
   - Клиент, столик, количество персон
   - Время начала и окончания (timestamp with time zone)
   - Проверка пересечений временных интервалов для одного столика
   - Проверка соответствия количества персон количеству мест

5. **ReservationDish (Позиции предзаказа)**
   - Связь между бронированием и блюдом с указанием количества
   - Уникальность пары (бронирование, блюдо)
   - Проверка доступного количества блюда

## Ограничения целостности данных

- Email клиента уникален
- Количество мест столика > 0
- Количество персон > 0 и ≤ количеству мест столика
- Дата окончания бронирования > даты начала
- Запрещены пересекающиеся бронирования одного столика (проверка на уровне модели и БД)
- Количество блюда в предзаказе > 0 и ≤ доступному количеству
- Все временные метки хранятся с учетом часового пояса

## Административная панель

### Управление клиентами
- Просмотр списка с поиском по ФИО и email
- Просмотр истории бронирований клиента
- Фильтрация по дате создания

### Управление столиками
- Просмотр и редактирование вместимости
- Отображение текущего статуса (занят/свободен)
- Фильтрация по количеству мест

### Управление блюдами
- Просмотр списка с изображениями
- Редактирование доступного количества
- Поиск по названию и описанию

### Управление бронированиями
- Создание и редактирование бронирований
- Добавление блюд в предзаказ через инлайн-формы
- Фильтрация по дате, столику, клиенту
- Защита от сохранения некорректных данных
- Отображение статуса бронирования (запланировано/активно/завершено)

## Важные замечания

- Для работы exclusion constraint в PostgreSQL требуется расширение `btree_gist`
- При удалении бронирования автоматически удаляются связанные позиции предзаказа (CASCADE)
- Все валидации выполняются как на уровне модели (Django), так и на уровне базы данных
